# Template 03 — Build-Method Landing Page

A single-page web page that walks visitors through how an instrument gets built. Public-facing on the maker site or the GitHub Pages of the instrument repo. Tells the story without giving away every drawing.

## Format

- **Single-page HTML** with anchor sections.
- **Hero band:** instrument name, mark, tagline, hero photo.
- **Story sections:** Origin → Materials → Tools → Method (3–5 phases) → Validation → Result.
- **Visual rhythm:** alternates text blocks and full-width process photos.
- **Footer:** maker bio + repo link + Heifer Zephyr colophon.

## Files this template consumes

Same set as the brochure (Template 01) plus:

| Source file | Used for |
|---|---|
| `study/README.md` | Validation/DoE story section |
| `images/` | Process photos throughout (not just hero) |
| `cnc/`, `cad/` if present | "Tools" section content |

## Voice

Editorial-narrative. Tells the build as a story, not a manual. First-person where Tony is the subject. Concrete materials and dimensions. No marketing puffery.

## What "done" looks like

- Reads start-to-finish in 3–4 minutes.
- Every section has at least one process photo.
- Repo URL appears in the footer + at the end of the body copy.
- Mobile-readable (single-column at narrow widths).
- No purchase CTA — this page is about the build, not the sale (Template 04 handles the sale).
